# 古神道v3 Phase 2 設計提案: 系列社キュレーションの収集方法

- 提出: こどたん → くろたん承認待ち(承認後に実行・指令書P2準拠)
- 日付: 2026-07-04

---

## 1. ゴールの再確認(指令書より)

- 規模: 都道府県ごとに各系列1〜3社 × 10系列 ≒ **500〜1,500社**
- 必須フィールド: 都道府県・緯度経度・lineage・出典タグ
- ソース制限: 神社公式サイト・神社本庁関連・自治体/観光公式のみ。**癸(瀬織津姫)は最厳格**
- 証拠: サンプル抽出20件+ソースURL一覧

## 2. 収集パイプライン設計(3段構え)

### Stage A: 既存資産+系統Aのシード統合(API不使用・コスト0)

- 既存 `data/kojindo/shrines/` の**112社(全て緯度経度付き)**を新スキーマに変換し、主祭神からlineageを付与
- 系統A(ひでさんDeep Research)の結果から総本社+代表系列社を取り込み
- 一の宮43社・別表神社29社は「格の高い系列社」として優先収載

### Stage B: web searchバッチ収集(メイン)

**単位**: (系列 × 都道府県) = 10系列 × 47都道府県 = **最大470クエリ**(充足済みセルはスキップ)

**実装**: Claude API + サーバーサイドweb searchツール + 構造化出力(JSON Schema強制)

```
モデル: claude-sonnet-5(収集・抽出は機械的作業のためSonnetで十分。
        単価はFableの1/3以下。品質判定は Stage C でFableが担当)
ツール: web_search_20260209(max_uses=5/クエリ)
出力: strict JSON Schema(name/pref/city/lat/lng/lineage/main_deity/
      source_url/source_type の全フィールド必須)
```

**プロンプトの骨子**(承認後に全文提出):
- 「{都道府県}にある{系列名}(御祭神: {神名})の神社を最大3社。**神社公式サイト・神社本庁・自治体/観光公式ページに記載が確認できるもののみ**。各社に出典URL必須。確認できない場合は0件で返す(推測禁止)」
- 癸のみ追加制約: 「主祭神として瀬織津姫を祀ることが公式由緒に明記されている神社のみ。配祀・合祀・俗説は除外」

### Stage C: 機械検品+Fable QA

1. **ドメイン許可リスト照合(機械・全件)**: 出典URLを正規表現で検査
   - 許可: `*.jinjahoncho.or.jp` / `*.lg.jp` / `pref.*.jp` / `city.*.jp` / 各神社公式ドメイン(Stage Bで採取したドメインをくろたん承認リスト化)
   - 拒否: アメブロ・note・まとめ系・スピリチュアル系(拒否リスト明文化)
   - **不合格レコードは自動破棄**(修正せず再収集)
2. **重複排除(機械)**: (正規化名称, 都道府県) キーで既存112社+新規を統合
3. **Fable 5によるサンプルQA**: 全体から10%を無作為抽出し「出典URLと記載内容の整合」を照合(指令書証拠のサンプル20件はここから提出)

## 3. データ格納設計

- `data/kojindo/shrines/lineage_{god_id}.json` × 10ファイル(系列単位・凍結god_id準拠)
- スキーマ = 既存スキーマ + P1拡張(`god_id` / `lineage` / `worship_override` / `is_primary` / `source` オブジェクト`{url, type, checked_at}`)
- 既存4ファイルは温存(v1構造のまま)。v3はlineageファイルを正とし、既存分はStage Aで変換コピー

## 4. コスト・工期見積り

| 項目 | 見積り |
|---|---|
| Stage B API費 | 470クエリ × (入力~6K+出力~1.5K tok, Sonnet 5) ≒ **$25〜40** + web search課金(470×5回=~2,350検索 ≒ $24) |
| Stage C QA費 | Fable 5で~100件照合 ≒ $5〜10 |
| 合計 | **$55〜75(約1万円)** |
| 工期 | 承認後、Stage A: 半日 / Stage B: 1日(レート制限次第) / Stage C: 半日 |

※Stage BをBatches API(50%オフ)にできればAPI費は半減しますが、web searchツールのバッチ対応可否を実行前に小規模テスト(5クエリ)で確認し、不可なら通常APIで実施します。

## 5. 承認いただきたい点

1. モデル分担(収集=Sonnet 5 / QA=Fable 5)と概算予算 $55〜75
2. ドメイン許可/拒否リストの初版(Stage B実行前に具体リストを提出→承認)
3. 「確認できない場合は0件」原則(空白セルを許容し、埋まらない県は無理に埋めない)
