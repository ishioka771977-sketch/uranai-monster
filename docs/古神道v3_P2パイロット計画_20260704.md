# 古神道v3 P2パイロット計画(浅間系)+ドメインリスト初版

- 提出: こどたん → くろたん承認待ち(承認後にパイロット実行)
- 日付: 2026-07-04
- 前提: くろたんP2条件1(パイロット先行・浅間系推奨)準拠

## 1. パイロット仕様

| 項目 | 内容 |
|---|---|
| 対象系列 | 浅間系(乙・木花咲耶姫・`konohanasakuya`)——全国約1,300社で母数最大 |
| クエリ数 | 47都道府県 × 1クエリ = **47クエリ**(全量470の1/10) |
| モデル | claude-sonnet-5 + web_search(max_uses=5/クエリ)+ strict JSON Schema |
| 目標収量 | 47〜141社(県あたり1〜3社・0件許容) |
| 検品 | ドメインリスト照合(全件・機械)→ 重複排除 → Fable 5で整合QA(20件抽出) |
| くろたん提出物 | サンプル20件(名称/県/緯度経度/出典URL/出典種別)+ 収量・棄却率レポート |
| 概算費用 | API $3〜5 + web search $2〜3 = **$5〜8** |
| 判定観点(提案) | ①出典がリスト準拠か ②実在神社か ③浅間系として正しいか(主祭神に木花咲耶姫)④緯度経度の妥当性 |

パイロットで検証したい仮説: (a)公式ソースだけで県1〜3社を充足できるか (b)棄却率がどの程度か(高すぎればプロンプト・許可リストを調整して残り9系列へ)

## 2. ドメイン許可/拒否リスト初版

### 自動合格(Tier 1: 疑義なく公式)

| パターン | 対象 |
|---|---|
| `jinjahoncho.or.jp` | 神社本庁 |
| `*jinjacho*.or.jp` / `*jinjyacho*` | 都道府県神社庁(例: 東京都神社庁 tokyo-jinjacho.or.jp) |
| `*.lg.jp` / `pref.*.jp` / `city.*.jp` / `town.*.jp` / `vill.*.jp` | 自治体公式 |
| `*.go.jp` | 政府機関(文化庁の文化財DB等) |
| `*.ac.jp` | 学術機関 |
| 都道府県観光連盟の公式ドメイン | 個別列挙表を実行前に添付(例: visit-hokkaido.jp 等) |

### 条件付き合格(Tier 2: 神社公式サイト)

神社の公式ドメインは形式がバラバラ(`.or.jp`/`.jp`/`.com`混在)のため、パターン照合では判定不能。以下の2条件を**両方**満たす場合のみ合格:
1. web search結果のタイトル/スニペットに「当該神社の公式サイト」であることが示されている(神社名+由緒/御祭神ページ)
2. Fable 5 QAで「そのドメインが当該神社自身のサイトである」ことを確認(パイロットでは20件全数、本番では10%抽出+Tier2全件の簡易チェック)

パイロットの副産物として**Tier 2で合格した神社公式ドメインを追補リスト化**し、くろたん承認を経て本番のTier 1に昇格させる(回すほど照合が固くなる設計)。

### 自動棄却(Tier 3)

`ameblo.jp` / `note.com` / `hatenablog` / `fc2` / `livedoor` / `wikipedia.org` / `hotokami.jp` / `omairi.club` / `goshuin`系まとめ / `matome`系 / 個人スピリチュアル系(キーワード: 波動・パワースポット認定・龍神様が〜等をURL/タイトルに含む媒体) / その他Tier1・2に該当しない全ドメイン

**デフォルトは棄却**(許可リスト方式)。「怪しいものを弾く」のではなく「確認できたものだけ通す」。

### 癸(瀬織津姫)特則——本番時

Tier 1のうち神社本庁・学術・自治体のみ+神社公式(Tier 2)は**Fable QA全数**。観光連盟は不可(伝承の混入リスク)。

## 3. 実行手順(承認後)

1. 小規模疎通テスト: 5クエリ(北海道・静岡・山梨・京都・沖縄)でBatches APIのweb search対応を確認 → 不可なら通常APIに切替
2. 47クエリ実行 → 機械検品 → QA → `data/kojindo/shrines/lineage_konohanasakuya.json`(パイロット版)生成
3. サンプル20件+収量レポートをくろたんへ提出 → 合格後、残り9系列へ展開(このとき系統A結果を許可リストへ反映)
