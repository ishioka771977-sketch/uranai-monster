## 第1部: 4エンジン読み比べ(実施済み・報告)

### 方式
- 対象エンジン: Claude Fable 5(現行) / Claude Opus 4.8 / GPT-5.6 Sol / Gemini 2.5 Pro(旧エンジン)
- **アプリが本番で組む実プロンプトを同一投入**(システム+本文をコードから捕獲)
- ブラインド(A〜D・ラウンドごとに並び替え)・ひでさん官能評価
- 2セット実施: ①固定命式(1990-05-20生) 7占術 ②**ひでさん本人命式** 7占術(1位+最下位方式)
- 費用: Anthropic分のみ(数百円)。GPT-5.6はcodex CLI(ChatGPT定額)、GeminiはAPIキー流用で追加費用ゼロ

### 結果(計14ラウンド)

| | 1位回数 | 最下位回数(本人版のみ集計) |
|---|---|---|
| Claude Fable 5 | 4.5 | 4 |
| Claude Opus 4.8 | 3.5 | 0 |
| Gemini 2.5 Pro | 4 | 2 |
| GPT-5.6 Sol | 2 | 1 |

### 所見
1. 絶対王者なし。同一エンジンの回ごとのブレが占術間の差より大きい
2. **Fable=高低差の激しい天才型**(勝ちは質、負けは全部「薄い」)。**Opus=最下位ゼロの安定型**
3. Geminiの敗因の典型: 万象学でエネルギー221を実際より過大な水準として記述(**数値の相場観の狂い**)
4. GPT-5.6は導入見送り(選ばれる回が少なくAPI新規契約の理由なし)

### 決定(ひでさん)
**エンジンはFable 5続投。**「薄い」問題は第2部で根治したため、Fableの弱点は解消済みと判断。

## 第2部: 「薄い」の根因分析と文章量床の実装(実施済み・報告)

### 根因の特定(3段階の実測)
1. Fableの出力は毎回1,300〜1,400字で4エンジン中最短。「薄い」評価は文章量とほぼ一致
2. ベースプロンプトに旧指定「1,200〜2,000文字」が残っており、**指示に忠実なエンジンほど下限に張り付く**構造
3. さらにコース側のv3原則「長さ自体は目標にしない」がベース指定を上書き。指定を1,600〜2,000に強化しても、Fable/Opusとも1,300〜1,550で不安定(**プロンプト指定だけでは字数は揃わない**ことを3回の実測で確認)

### 実施内容(PR #9・マージ済み)
- 分量条項の更新: ベース(目安1,800字・1,600未満は失敗扱い・上限2,200)+各コースJSON仕様+万象学(新設)
- **コード側の下限保証** `_call_api_length_floor`: 生成後に実測し1,600字未満なら「N字で短すぎた・密度を保って1,800字前後に」で書き直し1回
- 適用範囲: 7コース(算命学/占星術/九星/数秘/紫微/四柱/万象学)+テーマ深掘り5種+相性+コンプリート+堕ちるとき
- 対象外(意図的): **古神道(くろたん設計領域につき不触)**・タロット(ライト設計)・くろたんに聞く(Q&A)・開運暦(短文)
- 実測: Fable数秘術1,434→1,594字 / 算命学1,410→1,628字 / 恋愛運1,519→1,577字(発火ログつき)
- コスト影響: 初回が短い場合のみ書き直しが走る(最大2倍/コース)

### ⚠️ くろたんへの申告事項
本実施は**v3第1原則「文字数の上限・下限は設けない」の部分的な巻き戻し**を含む(ひでさん指示)。
根拠データ: ブラインド評価で「薄い」判定が短出力と一致(本人版でFable最下位4回全て)。ひでさん再評価では床適用後「かなり良くなった」。
→ **諮問①: この方針変更の追認をお願いしたい。**

### 副次発見
- 旧(短い)版のほうが一文の切れ味は上のケースあり(「速い体と遅い言葉。この時間差が説得力の正体」等)。床は「満腹感を保証し、切れ味はモデルの地力に任せる」分担
- 紫微斗数のみ床でも1,378字止まり → 第3部とは別に、**知識供給の問題**と診断:
  - 万象学プロンプト8,337字=知識8割(書籍1冊スキャンの成果)
  - 紫微斗数プロンプト5,038字=書き方8割・知識2割(副星・格局・廟旺落陥の教義が無い)
  - 四柱推命が最も濃い理由=コード計算(格局・神殺・用神)+教義レイヤー+モデル地力の三重供給
  → **諮問②: 紫微斗数の書籍KB化**(『紫微斗数実占ハンドブック』等2冊を book-kb-pipeline で知識ベース化→プロンプト知識層を構築)。実施時は鑑定プロンプト大改変になるため設計段階で関所に諮る。
  リソース見積もり(新ルール準拠): 書籍2冊×約300頁のスキャン読取(Sonnet 5)+品質審査(Opus 4.8)で概算数百回のAPI呼び出し・web検索ゼロ

## 第3部: 相性占いスコアロジック内部調査(調査のみ・未改変)

### 調査の発端(ひでさん)
「点数の付け方が万象学(エネルギー差)に強く影響される。ひでさん(342)は大抵の相手と点差が大きく低得点になりがち。逆に342×342なら高得点になるはずだが、超強力×超強力が良い相性とは思えない」

### ロジックの実態(core/aisho_scoring.py)
- スコア=5項目(日干五行/エネルギー差/第1本能/天中殺/星座)の関係性別加重平均×10
- エネルギー項は**差の絶対値のみ**: 差30以内=10点 / 60以内=8 / 100以内=5 / 150以内=3 / **150超=1**
- 合計値・絶対水準の概念はコード上に存在しない

### 検証結果(仮説は2つとも正)
1. **ひでさん(342)の構造的失点**: 平均的な相手(190前後)との差152→1/10。100点換算の自動失点は 上司部下▲36 / 親子▲32 / 結婚▲21 / 恋愛▲14。上司部下・親子は配点の4割がエネルギー差
2. **342×342は差0でエネルギー項満点**。高×高の衝突リスクはスコア上むしろ最高評価
3. ただし別軸に質的警告は存在: 第1本能が攻撃×攻撃だと「ブレーキ役がいない」テキストが出る(がスコアは7/10で減点なし)。**文章は知っているのに点数が知らない**状態
4. 教義との整合: コードは万象学教義(差30/60/100、高い方が外で消費)に忠実=バグではなく設計どおり。**「高×高ペア」は教義側の空白地帯**(スキャン済み知識にも記述なし)

### 諮問③: 設計判断をお願いしたい(選択肢)
- 案A(最小): 両者250超なら差0でも満点にせず8点+「ブレーキ不在」注記を追加
- 案B(本格): エネルギー項を「差×合計」の2次元マトリクス化
- 案C: 上司部下/親子のウェイト3.0の見直し
- 案D: 点数は不変、解釈ラベル・文章側で吸収
- 補足: 書籍の相性章に高×高の記述があるか、紫微斗数KB化と同時に万象学書籍も再確認可能
