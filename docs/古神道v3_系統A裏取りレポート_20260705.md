## 4. 瀬織津姫(癸)——全神格で最も固い土台が完成

- **佐久奈度神社(滋賀県大津市)が唯一の確定社**: 公式由緒+滋賀県神社庁DB(配祀神と明確区別)+大津市歴史博物館(七瀬祓所)の3層一次情報
- 制度史: 天智天皇8年(669)中臣金連による祓創始の由緒、式内名神大社(延喜式神名帳)、六国史の位階記録、平安期の七瀬祓所
- 原典: 大祓詞=『延喜式』巻八「六月晦大祓」が文献上の初出(國學院大学解説+査読論文)。瀬織津比咩の役割=罪を川から大海原へ運ぶ第一段階
- **強力な逆証拠**: 明治5年教部省達第7号の国定祓詞に「瀬織津姫神」筆頭明記(國學院リポジトリ・一次史料翻刻)——「明治政府が抹消した」説と正面から矛盾
- マスタ表現の注意: 公式に「主祭神」の語はなく**「祓戸四神を主祭神格で奉斎(瀬織津姫筆頭)」**が正確。「総本社」表記は不可
- 意外な接続: **出雲大社の祓社の祭神が祓戸四柱(瀬織津比咩神筆頭)**と公式摂末社ページに明記——「まず祓社で祓い清める」出雲の公式順路と癸の神格が繋がる(鑑定文の素材として上質)

## 5. P2パイプラインへの技術教訓(パイロット設計に反映)

1. **Shift_JIS幻覚**: WebFetch要約が滋賀県神社庁DB(Shift_JIS)で幻覚を起こした事例を検証側が検出。**検品は生HTML取得+文字コード復号+逐語照合を必須とする**
2. **HTTP-onlyサイト**: 志賀海・浅間・佐久奈度など公式がHTTP接続のみ(HTTPS不可)——収集ツールのHTTPSアップグレード強制に注意
3. **URL脆弱性**: 神社庁DB(CGI形式)は将来切れやすい→**マスタに取得日+引用文そのものを保存**する仕様(出典タグに quote/retrieved_at フィールド追加)
4. 公称数字(1,300/2,300/3,000/239等)は全て**神社側の自称+公的追認**であり独立集計ではない→マスタでは出典種別を「公称」と明記

## 7. くろたん確定事項(2026-07-05 レビュー回答・以下は確定仕様)

1. **丁のis_primary = 月讀宮(伊勢別宮)で確定**。月読神社(京都)は `is_primary=false` で収載継続し、関西圏レコメンドの受け皿とする。物語上の採用点: 丙(伊勢)と丁(月讀宮)が同じ神域に並ぶ「太陽と月の兄弟神」構図は十干の陽火・陰火の対に照応
2. **社数表示は「約+公称」原則で確定**。UI表示は「全国約2,300社(公称)」形式。公称値が存在しない柱(甲・丙の神明系等)は社数非表示とし「◯◯系の神社が各地に」という定性表現に逃がす
3. **`self_designation` フィールドをマスタに追加**。総本社/総本宮/日本総鎮守/名乗りなし——**公式の名乗りに忠実に**格納・表示し、アプリ側で勝手に「総本社」を付与しない(瀬織津姫の表記原則を全柱に一般化)
4. **天の逆鉾は霧島神宮の描写での使用禁止**。辛の物語素材は「高千穂峰への天孫降臨」まで。逆鉾に言及する場合は「霧島東神社の社宝」と明記(現地での違和感=アプリの信頼問題)
5. **伊勢の意識の種はくろたん改訂版に差し替え(骨格版更新)**:
   > 「願う前に、照らされていることに気づく」——正宮では日々への感謝だけを述べる。特別な願いがあるなら、神楽殿でのご祈祷という正式な作法へ。
6. **祓社クロスリンク構造を採用**。「あなたの守護神の社で、最初に出迎えるのは別の柱の神」(出雲×瀬織津姫等)を、10柱が一つの神統譜であることを示すv3鑑定文の語りとして採用(P3プロンプト設計に反映)
