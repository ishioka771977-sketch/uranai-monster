```

├── stories/          ← 物語ストック（層1）
│   ├── kojiki.json

│   ├── miyashita.json    （v1.7以降）

│   └── v1_seed.json      ← v1の固定マッピング10件を継承
├── shrines/          ← 神社DB（層2・古神道+開運習慣L3-L5で共有）
│   ├── major.json    （一宮・別表神社・名神大社）
│   ├── regional.json （地方の特殊社）
│   └── deity_index.json  ← 祭神→神社IDの逆引き（自動生成可）
└── tags/             ← タグ辞書
    └── tag_dictionary.json   ← 30-50個に絞る（くろたんが初期20個提供）
```

## JSONスキーマ（決定版）

### Story（物語エピソード）

```json
{
  "id": "kojiki_kuzuryu_01",          // ユニークID（出典+主題+連番）
  "title": "九頭龍の調伏",             // 短いタイトル
  "source": "箱根神社縁起",            // 出典の具体名
  "source_type": "神社縁起",           // カテゴリ（古事記/日本書紀/宮下文書/竹内文書/風土記/神社縁起/漫画/映画）
  "trust_level": "古史古伝",           // 信頼度（正史/古史古伝/現代物語）
  "god": "九頭龍神",                   // 主役の神
  "story_summary": "芦ノ湖を荒らした毒龍が、万巻上人の法力で調伏され龍神として祀られた",
  "story_one_line": "毒龍が龍神に変わった。喰えば味方になる",  // CMプランナー型1行
  "tags": ["火の制御", "規格外エネルギー", "毒→薬の変換"],   // 説明タグ（自由）
  "applicable_traits": ["エネルギー300超", "火過多", "用神水"],  // ★マッチング用タグ（tag_dictionary.json 準拠）
  "shrine_id": "hakone_kuzuryu"        // 紐づく神社ID
}
```

## 運用ルール

1. **タグの追加は月1回レビュー**（タグ膨張防止・くろたん+コドたん）
2. **エピソード作成時に `applicable_traits` は tag_dictionary.json から選ぶ**（独自タグ禁止）
3. **出典は必ず明記**（UI でも表示する）
4. **trust_level に応じて鑑定文の表現を調整**
   - 正史: 「神話に〜とある」
   - 古史古伝: 「ある伝承では〜と語られる」
   - 現代物語: 「物語の中の〜のように」
